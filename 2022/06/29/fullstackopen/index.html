FullStackOpen 2022 心得（一）

Posted on 2022-06-29 In career

關於課程


這是赫爾辛基大學（可以說是芬蘭台大）的公開課程，副標題是Deep Dive Into Modern Web Development，以下是課程連結：
https://fullstackopen.com/en/
可以在課程首頁看到能學到的技術，前端主要是React，後端是Node.js+MongoDB。課程從入門的前後端，再到深入的前後端，以及其他畫龍點睛的章節如TypeScript、CI/CD、Container等等，總共分為14個部分，每個部分除了範例程式碼的實作之外，也有不少量的文字做註解，也會附上一些reference的blog文章討論來說明為何選擇範例那樣的實作方式。而在實作後也都會有練習題，而官方也有提交系統讓大家上傳練習題repo，特別的是完成了每個課程習題的人可以直接獲得課程作者的公司的面試機會（但必須是芬蘭人），如果是外國人的話依然可以在完成核心課程(Parts 0-5)足夠數量的練習題後下載課程證書。

課程內容


由於我自己還只完成前幾部分的課程，就先對我已經完成的內容作簡單介紹：

Part 0 - Fundamentals of Web apps


這個Part總共又分為兩小節，第一是更詳細的課程內容，包括前一部分所說的學分的計算方式以及證書的取得方式、練習作業的提交方式、以及推薦的開發環境等等(課程推薦使用Chrome瀏覽器以及VSCode作為Editor)，第二是介紹Web架構從傳統到現代前後端分離的演進、HTTP的request methods介紹，以及教大家在瀏覽器按F12來觀察這些動作。

Part 1 - Introduction to React


分為四個小章節，第一章是說明React專案的架構，包括JSX和Component及傳遞props的運作方式。第二章是Javascript的介紹，講解陣列、物件、Arrow Function的寫法等等。第三和第四主要都在介紹React的Hook(useState)跟Event Handling的各種寫法以及不同寫法的注意事項，以及一些重構組件的思考方式。

這個章節正式開始進入React開發的部分，身為過去只有寫過古板的HTML網站並用jQuery做event handling的菜鳥前端，第一個感想是在JSX寫大括號塞變數的動態渲染也太方便了！但接下來的小難關就是要理解useState的運作方式，不過如果有詳細閱讀文件應該是沒問題的，再來比較困難的就是每個組件的切分方式、要切分得多細、Function拉出來寫還是直接寫在大括號裡，這些我猜只能靠經驗的累積了，多看多學多問。

Part 2 - Communicating with server


分為五個小節，前兩小節主要時介紹陣列的map、reduce、filter等方法，結合JSX的特性及useState的應用，以達到新增物件到陣列時前端畫面也跟著做改變的效果。後兩個小節則是使用json-server當成假後端，並介紹REST, axios, promise, useEffect等等的概念對後端丟請求以及設定接下來的動作，以及將axios切出一層service module的常見重構方式。最後則是CSS在JSX中的寫法，例如class要寫成className這點。

到這裡已經能學到前後端分離的概念了，利用axios跟後端要資料再render回前端，對比過去的經驗大概就是jQuery的ajax function的類似用法，課程文檔裡也有補充Promise是現代JavaScript開發的重點，可以花時間好好理解，不過現在的我還是以興趣學習為主，未來如果需要以JS做為主要開發語言的時候應該會再補看。

Part 3 - Programming a server with NodeJS and Express


這個部分分為四小節，第一小節是Node.js和Express的介紹，以及介紹一些方便開發用的套件例如nodemon，以及小工具如Postman，還有一些logger的middleware的用法講解。第二部分是前後端的整合以及部屬到Heroku的準備，例如需要注意CORS和設定proxy，還有build出來的前端檔案需要丟給express的static。第三部分則是MongoDB的介紹，課程中使用MongoDB Atlas的線上服務來做範例還挺方便的。第四部份則是Heroku上的App連線到DB的設定以及ESLint的檢查。

課程到現在已經算是學會了整包前後端的雛型，我猜以這樣的能力來說應該足以應付大部分entry level的工作職責了（有誤歡迎指正），回想過去我的第一份工作差不多就是在做這些事，在公司放任讓我自己摸索的狀況下，可能過了一個月才達到這種程度，然而現在花不到一週的時間學到現在的進度了，而且還是較為現代的開發方式，不禁替太晚發現這個課程感到些許可惜。

學習心得


我自己是覺得這些課程的內容，相比過去上課所學的所有課程都更接近實務，其實我不清楚其他學校的資工系或資管系有沒有專門介紹網頁開發的課程，但如果不是自發性地做Side Projects或是想接案練功找實習的話，在大學四年內可能都不會有接觸到Web開發知識的機會（本人就是這樣），雖然網路上有超多網頁開發相關的資源，只要有心一定能學得會，但我覺得有個引路人一定能更有效率地學習，甚至能帶出學生對相關領域的興趣，而這們課程作為引路人再適合不過了。

原本只是想在等待研究所開學的期間找找看有沒有可以殺時間的material，如果找不到的話可能就是在leetcode刷刷題，但好在有讓我發現這個佛心課程。過去雖然有將近一年的工作經驗，但由於產業與職缺性質的緣故，雖然我能稱得上在做網頁開發的工作，但對比現代的網頁開發，或者更大範圍來說，對比現代的軟體開發方式，我真的算是落後許多了（例如前兩個東家都完全沒在用git）。

由於接下來就要搬去台北，準備研究所開學以及選指導教授的事情，課程就先停在這裡，暑假期間如果沒有什麼事情要忙的話應該還會繼續鑽研，希望這個課程心得不會只有停在（一）而已，跟NL的萬聖節影片一樣富監。
